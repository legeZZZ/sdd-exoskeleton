# SDD Change Proposal Prompt

Propose a change to the project following SDD workflow.

1. **Describe the change**: What problem does it solve? What modules does it touch?
2. **Check boundaries**: Does it cross module boundaries? If so, list affected APIs.
3. **Create proposal**: Write `openspec/changes/<name>/proposal.md`
4. **Get approval**: Have the proposal reviewed before implementing
5. **Implement**: Write code, run tests, update specs
6. **Archive**: Move completed change to archive
