# SDD Project Onboarding Prompt

Onboard a new developer to an SDD-enabled project.

1. **Read the CLAUDE.md**: Understand project overview, tech stack, module map
2. **Explore the Obsidian vault**: Open the vault in Obsidian, browse module notes
3. **Read key specs**: `openspec/specs/modules/` — start with core modules
4. **Check active changes**: `openspec/changes/active/` — what's in progress?
5. **Run sdd status**: See current project health
6. **Understand the workflow**: propose → apply → verify → archive
